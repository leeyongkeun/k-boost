# K-BOOST 코드 리뷰 (2026-03-15)

## 1. 코드 개선

### 보안
- **Rate Limiting 없음** — `/api/analyze` 호출 1회에 네이버+카카오+Google+Claude API 4개가 돌아감. 누군가 반복 호출하면 API 크레딧 소진. Vercel Edge Middleware로 IP당 분당 5회 제한 추가 권장
- **입력 검증** — `store_info`에 길이 제한이 없음. 200자 초과 차단 + 특수문자 필터 추가
- **`/api/result`** — 클라이언트가 보낸 결과를 검증 없이 DB에 저장 중. Zod 스키마 검증 추가 필요

### 버그
- **`analyze-with-data.ts`에서 `instaP!` 사용** — Instagram 플랫폼은 platforms 배열에 없는데 `find()!`로 접근 → undefined 접근 에러 가능
- **KST 변환이 수동** — `Date.now() + 9시간` 방식은 한국에 DST가 없어서 동작하지만, 코드 의도가 불명확

### 구조
- **claude.ts와 gemini.ts에 분석 로직 중복** — 점수 정규화, 결과 변환 코드가 3곳에 반복. `lib/normalize-result.ts` 하나로 추출하면 유지보수 편해짐
- **상수 흩어짐** — 타임아웃(6s, 14.4s, 18s), 캐시 TTL(7일) 등이 각 파일에 흩어져 있음. `lib/constants.ts`로 모으면 조정이 쉬움

## 3. 기획/비즈니스 추천

### 즉시 적용 가능
- **리드 수집 강화** — 결과 페이지에서 "상세 컨설팅 받기" CTA → 이름/연락처/이메일 입력 폼
- **결과 공유 기능** — 카카오톡/링크 공유 버튼. 바이럴 효과 + 자연 유입 증가
- **재검색 유도** — "다른 매장도 진단해보기" 버튼. 세션당 분석 횟수 증가

### 중기 (데이터 쌓인 후)
- **업종별/지역별 리포트** — 축적 데이터로 "홍대 카페 평균 점수 vs 내 매장" 비교 차트
- **경쟁 매장 비교** — 같은 동네 같은 업종 매장 2~3개 비교 분석
- **PDF 진단서 다운로드** — 결과를 PDF로 변환. 컨설팅 보고용
- **Before/After 추적** — 같은 매장 재검색 시 이전 점수 대비 변화 표시

### 장기
- **자동 정기 모니터링** — 등록 매장의 플랫폼 데이터를 주 1회 자동 수집 → 유료 구독 모델
- **다국어 진단** — 일본/대만 사업자용 영어/일어 버전
